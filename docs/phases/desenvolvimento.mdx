## Objetivo
Implementar mudanças de forma incremental e sincronizada, reduzindo conflitos acumulados por meio de commits atômicos, sincronização frequente e validações locais.

## Entradas (Required Artifacts)
- [Branch de Atividade](/docs/artifacts/branch-atividade)
- Convenção de mensagens de commit estabelecida

## Saídas (Produced Artifacts)
- [Commits Padronizados](/docs/artifacts/commits-padronizados)
- Branch atualizada
- Evidências de testes locais

## Papéis
- **[Developer A](/docs/roles/developer-principal)** — implementa mudanças, realiza commits atômicos e sincroniza branch
- **[Developer B](/docs/roles/developer-integrador)** — co-responsável pela implementação e sincronização

## Atividades
1. [Implementar Mudanças](/docs/activities/implementar-mudancas) com commits atômicos (1 mudança lógica por commit)
2. Sincronizar branch com `main` regularmente (merge/rebase periódico)
3. Executar testes locais

## Critérios de Entrada
- Branch criada, escopo definido e convenção de commits estabelecida

## Critérios de Saída (Gate)
- Testes locais aprovados; branch atualizada; commits padronizados

## Quality Gate
**[Commit Message Convention](/docs/guidelines/commit-message-convention)** — todas as mensagens de commit seguem o padrão definido antes de avançar para a fase de Pull Request.
