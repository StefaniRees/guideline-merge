## Objetivo
Preparar condições para um merge seguro: definir branch, escopo e convenções; reduzir risco de conflitos com atualização contínua.

## Entradas (Required Artifacts)
- [Ticket/issue](/docs/artifacts/branch-atividade) definido e priorizado
- Convenção de branch e commits estabelecida

## Saídas (Produced Artifacts)
- [Branch de Atividade](/docs/artifacts/branch-atividade)

## Papéis
- **[Developer A](/docs/roles/developer-principal)** — cria branch, define plano de sync, realiza commits padronizados
- **[Developer B](/docs/roles/developer-integrador)** — co-responsável pela branch e sincronização
- **[Mediador](/docs/roles/mediador)** — valida convenções e sinaliza riscos

## Atividades
1. [Criar Branch](/docs/activities/criar-branch) a partir de `main`/homologação (`feature/`, `bugfix/`, `hotfix/`)
2. Definir escopo e responsáveis
3. Atualizar frequentemente a branch com a principal (merge/rebase periódico)
4. Realizar commits padronizados (pré-PR)


## Quality Gate
**[Branch Naming Convention](/docs/guidelines/branch-naming-convention)** — branch válida com prefixo, ID do ticket e plano de sincronização registrado antes de avançar para a fase de Desenvolvimento.
