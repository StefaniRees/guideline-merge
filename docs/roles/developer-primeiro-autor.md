## Descrição
Atua como **autor inicial da mudança**. É responsável por conduzir a implementação de uma tarefa prioritária ou mais complexa, criando a branch de atividade de referência.  
Além de implementar, testa localmente e abre o **Pull Request inicial**, que servirá como base para integrações posteriores. Assume papel ativo na **resolução de conflitos**, pois sua branch tende a gerar o impacto principal no merge.

## Responsabilidades
- Implementar tarefas principais ou críticas
- Criar e manter a branch de referência
- Garantir rastreabilidade (ticket ↔ commits ↔ PR)
- Resolver conflitos que envolvem seu código como base
