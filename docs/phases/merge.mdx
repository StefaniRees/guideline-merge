## Objetivo
Integrar a branch na `main`/master somente após PR aprovado e checks “verdes”. Conflitos resolvidos preferencialmente de forma colaborativa.

## Entradas
- PR aprovado; branch sincronizada com `main`

## Saídas
- Merge commit/squash executado
- Build de integração pós-merge (smoke)

## Papéis
- **Mediador (A/R)** — coordena merge e resolução de conflitos
- **Desenvolvedores (R)** — apoiam resolução quando necessário

## Atividades
1. Resolver conflitos de merge (colaborativo quando crítico).
2. Executar build pós-merge (smoke).

## Critérios de Entrada
- Aprovação formal; CI verde

## Critérios de Saída
- Smoke pós-merge ok; integração auditada e rastreável
