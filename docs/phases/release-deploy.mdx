## Objetivo
Promover a versão integrada; comunicar mudanças; monitorar o sistema.

## Entradas
- CI/CD de release configurado

## Saídas
- Release implantada; Release Notes publicados; evidências de monitoramento

## Papéis
- **Dev/DevOps (A/R)** — pipeline e deploy
- **Mediador (C)** — valida comunicação e evidências

## Atividades
1. Gerar Release Notes (preferência automática via commits/PRs).
2. Realizar deploy via pipeline (sem passos manuais críticos).
3. Monitorar pós-deploy (observabilidade/alertas).

## Critérios de Entrada
- Merge concluído; artefatos prontos

## Critérios de Saída
- Sistema estável; stakeholders informados
