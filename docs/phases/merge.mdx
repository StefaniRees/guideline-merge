## Objetivo
Integrar a branch aprovada à principal de forma controlada e auditável, resolvendo conflitos de forma colaborativa e consolidando a versão integrada.

## Entradas (Required Artifacts)
- [Pull Request](/docs/artifacts/pull-request) aprovado
- [Relatório de QA](/docs/artifacts/relatorio-qa)
- Branch sincronizada com main

## Saídas (Produced Artifacts)
- Merge commit ou squash
- Build de integração
- [Histórico de Conflitos](/docs/artifacts/historico-conflitos)
- [Commits Padronizados](/docs/artifacts/commits-padronizados) de resolução
- [Release Notes](/docs/artifacts/release-notes)

## Papéis
- **[Mediador](/docs/roles/mediador)** — coordena a integração, apoia resolução de conflitos e garante rastreabilidade
- **[Developer A](/docs/roles/developer-principal)** — apoia resolução de conflitos quando necessário
- **[Developer B](/docs/roles/developer-integrador)** — apoia resolução de conflitos quando necessário

## Atividades
1. [Resolver Conflitos de Merge](/docs/activities/resolver-conflitos) — identificar e resolver conflitos entre branch e main
2. Registrar causas, decisões e histórico de resolução
3. [Merge Final](/docs/activities/merge-final) — integrar branch na main/release após aprovações
4. Consolidar versão integrada e gerar Release Notes
5. Executar build de integração

## Critérios de Entrada
- PR aprovado; Relatório de QA disponível; pipeline CI/CD aprovado

## Critérios de Saída (Gate)
- Conflitos resolvidos e registrados + build de integração aprovado + Release Notes geradas

## Quality Gate
**[Conflict Resolution Procedure](/docs/guidelines/conflict-resolution-procedure)** — conflitos identificados, resolvidos colaborativamente, com causas, decisões e histórico registrados antes de avançar para Release/Deploy.
